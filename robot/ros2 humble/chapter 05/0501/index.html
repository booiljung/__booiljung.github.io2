노드의 기본 개념

ROS2에서 노드(Node)는 독립적인 실행 단위로, ROS2 시스템에서 가장 중요한 구성 요소 중 하나이다. 노드는 센서 데이터를 수집하거나, 이를 처리하여 제어 명령을 생성하는 역할을 하며, ROS2 네트워크 상의 다른 노드들과 메시지를 주고받을 수 있다. 각 노드는 고유한 이름을 가지며, 동일한 이름의 노드를 중복해서 실행할 수 없다. 노드는 퍼블리셔, 서브스크라이버, 서비스, 액션 등을 포함하여 다양한 형태의 통신 인터페이스를 제공한다.

노드는 시스템 내에서 독립적인 작업을 수행하며, 필요에 따라 ROS2 네트워크 상의 다른 노드들과 데이터를 교환한다. 이를 통해 분산 시스템의 구조를 갖추고, 복잡한 로봇 시스템에서도 유연한 통신이 가능하게 한다.

노드의 생성

노드를 생성하기 위해서는 ROS2에서 제공하는 라이브러리를 이용하여 프로그램을 작성한다. ROS2는 C++와 Python을 지원하며, 각각의 언어에 맞는 API가 제공된다.

Python에서의 노드 생성

Python에서 ROS2 노드를 생성하기 위해서는 rclpy 라이브러리를 사용한다. 기본적인 노드 생성 과정은 다음과 같다.

import rclpy
from rclpy.node import Node

class MyNode(Node):
    def __init__(self):
        super().__init__('my_node')

def main(args=None):
    rclpy.init(args=args)
    node = MyNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()

이 코드에서 MyNode 클래스는 Node 클래스를 상속받아 생성된 노드이다. __init__ 메서드에서 노드의 이름인 'my_node'를 정의하며, rclpy.spin() 함수는 노드를 계속 실행 상태로 유지한다.

C++에서의 노드 생성

C++에서는 rclcpp 라이브러리를 사용하여 노드를 생성한다. 기본적인 노드 생성 과정은 다음과 같다.

#include "rclcpp/rclcpp.hpp"

class MyNode : public rclcpp::Node
{
public:
  MyNode() : Node("my_node") {}
};

int main(int argc, char * argv[])
{
  rclcpp::init(argc, argv);
  auto node = std::make_shared<MyNode>();
  rclcpp::spin(node);
  rclcpp::shutdown();
  return 0;
}

이 코드에서 MyNode 클래스는 rclcpp::Node 클래스를 상속받아 생성되며, Node 생성자에서 노드의 이름 'my_node'를 정의한다.

노드의 관리

노드를 생성한 후에는 다양한 방식으로 노드를 관리할 수 있다. 노드 관리는 크게 두 가지로 나뉜다: 노드 자체의 관리와 노드 간의 통신 관리이다.

노드의 초기화 및 종료

노드를 실행하려면 rclpy.init() 또는 rclcpp::init()을 호출하여 ROS2 시스템을 초기화해야 한다. 노드 실행이 완료되면 rclpy.shutdown() 또는 rclcpp::shutdown()을 통해 시스템을 안전하게 종료해야 한다.

노드 이름 관리

각 노드는 유일한 이름을 가져야 하며, 이를 통해 네트워크 상에서 다른 노드들과 구별된다. 노드 이름을 명시하지 않고 생성하면 자동으로 임의의 이름이 부여될 수 있다. 노드 이름은 노드를 생성할 때 전달되는 첫 번째 인자에 의해 설정되며, 동일한 이름의 노드가 실행 중인 경우 에러가 발생할 수 있다.

class MyNode(Node):
    def __init__(self):
        super().__init__('unique_node_name')

class MyNode : public rclcpp::Node
{
public:
  MyNode() : Node("unique_node_name") {}
};

노드의 수명주기 관리

ROS2는 노드의 수명주기(Lifecycle)를 관리할 수 있는 기능을 제공한다. 이를 통해 노드를 일시적으로 비활성화하거나 재활성화할 수 있으며, 이 기능은 실시간 시스템에서 매우 유용하게 활용된다. 노드의 상태는 총 4가지로 나뉘며, 각 상태 간의 전환이 가능한다.

노드의 수명주기 관리는 ROS2에서 제공하는 LifecycleNode 클래스를 사용하여 구현할 수 있다.

#include "rclcpp_lifecycle/lifecycle_node.hpp"

class LifecycleNodeExample : public rclcpp_lifecycle::LifecycleNode
{
public:
  LifecycleNodeExample() : LifecycleNode("lifecycle_node_example") {}

  // 상태 전환 관련 함수들...
};

이 예제에서는 rclcpp_lifecycle::LifecycleNode를 상속받아 수명주기를 관리하는 노드를 생성하였다. 수명주기를 활용하여 노드의 상태 전환, 일시 중단 및 재시작 등을 관리할 수 있다.

노드의 상태 전환 및 관리

ROS2의 수명주기 노드(Lifecycle Node)는 노드의 상태를 다섯 가지 상태로 나누어 관리한다. 각 상태는 특정 기능을 수행하거나 시스템의 요구 사항에 맞게 노드를 일시 중지하거나 재활성화할 수 있도록 한다.

수명주기 노드의 상태

  1. Unconfigured: 노드가 초기화되었지만 아직 구성되지 않은 상태이다. 노드가 구체적인 기능을 수행하기 전 상태이다.
  2. Inactive: 노드가 구성되었지만 활성화되지 않은 상태이다. 이 상태에서는 노드가 외부와 상호작용하지 않으며, 데이터를 보내거나 받지 않는다.
  3. Active: 노드가 완전히 활성화된 상태이다. 노드가 퍼블리셔, 서브스크라이버, 서비스 등을 통해 외부와 통신을 할 수 있다.
  4. Finalized: 노드가 종료된 상태이다. 이 상태에서는 노드가 더 이상 작동하지 않으며, 재활성화할 수 없다.
  5. Error: 노드가 예외적인 오류가 발생한 상태이다. 오류를 해결하지 않으면 노드를 다시 활성화할 수 없다.

상태 전환 함수

각 상태 간의 전환은 ROS2가 제공하는 수명주기 이벤트를 통해 이루어진다. 상태 전환 함수는 다음과 같다.

  • on_configure(): 노드가 Unconfigured 상태에서 Inactive 상태로 전환될 때 호출된다.
  • on_activate(): 노드가 Inactive 상태에서 Active 상태로 전환될 때 호출된다.
  • on_deactivate(): 노드가 Active 상태에서 Inactive 상태로 전환될 때 호출된다.
  • on_cleanup(): 노드가 Inactive 상태에서 Unconfigured 상태로 전환될 때 호출된다.
  • on_shutdown(): 노드가 종료될 때 호출된다.

이러한 상태 전환 함수들을 통해 노드의 상태를 명확히 관리할 수 있으며, 각 상태에서 특정 작업을 수행할 수 있다.

#include "rclcpp_lifecycle/lifecycle_node.hpp"
#include "rclcpp/rclcpp.hpp"

class LifecycleNodeExample : public rclcpp_lifecycle::LifecycleNode
{
public:
  LifecycleNodeExample() : LifecycleNode("lifecycle_node_example") {}

  rclcpp_lifecycle::LifecycleNodeInterface::CallbackReturn on_configure(const rclcpp_lifecycle::State &)
  {
    RCLCPP_INFO(this->get_logger(), "Configuring");
    return rclcpp_lifecycle::LifecycleNodeInterface::CallbackReturn::SUCCESS;
  }

  rclcpp_lifecycle::LifecycleNodeInterface::CallbackReturn on_activate(const rclcpp_lifecycle::State &)
  {
    RCLCPP_INFO(this->get_logger(), "Activating");
    return rclcpp_lifecycle::LifecycleNodeInterface::CallbackReturn::SUCCESS;
  }

  rclcpp_lifecycle::LifecycleNodeInterface::CallbackReturn on_deactivate(const rclcpp_lifecycle::State &)
  {
    RCLCPP_INFO(this->get_logger(), "Deactivating");
    return rclcpp_lifecycle::LifecycleNodeInterface::CallbackReturn::SUCCESS;
  }

  rclcpp_lifecycle::LifecycleNodeInterface::CallbackReturn on_cleanup(const rclcpp_lifecycle::State &)
  {
    RCLCPP_INFO(this->get_logger(), "Cleaning up");
    return rclcpp_lifecycle::LifecycleNodeInterface::CallbackReturn::SUCCESS;
  }

  rclcpp_lifecycle::LifecycleNodeInterface::CallbackReturn on_shutdown(const rclcpp_lifecycle::State &)
  {
    RCLCPP_INFO(this->get_logger(), "Shutting down");
    return rclcpp_lifecycle::LifecycleNodeInterface::CallbackReturn::SUCCESS;
  }
};

이 코드에서 LifecycleNodeExample 클래스는 각 상태 전환에서 로그 메시지를 출력하며, 성공적으로 상태가 전환되었음을 알린다.

노드 상태 전환 시의 예외 처리

수명주기 노드에서 상태 전환 중 발생할 수 있는 예외 상황을 처리하는 것이 중요하다. 예를 들어, 특정 하드웨어 장치와의 통신이 실패하거나 노드 내부에서 발생하는 오류가 있을 수 있다. 이런 경우 on_error() 콜백을 통해 오류를 처리할 수 있다.

rclcpp_lifecycle::LifecycleNodeInterface::CallbackReturn on_error(const rclcpp_lifecycle::State &)
{
  RCLCPP_ERROR(this->get_logger(), "Error occurred during state transition");
  return rclcpp_lifecycle::LifecycleNodeInterface::CallbackReturn::ERROR;
}

상태 전환을 통한 실시간 시스템 제어

수명주기 노드는 실시간 시스템에서 매우 유용하게 활용될 수 있다. 노드를 일시적으로 비활성화하거나 필요에 따라 상태를 전환함으로써 시스템의 자원을 효율적으로 관리할 수 있다. 예를 들어, 실시간으로 데이터를 처리하는 로봇 시스템에서는 노드를 활성화하여 필요한 작업을 수행하고, 이후에는 노드를 비활성화하여 자원을 절약할 수 있다.

수명주기 노드가 실시간 시스템에 미치는 영향을 최소화하기 위해 상태 전환은 비동기적으로 처리되며, 각 상태에서 수행하는 작업의 효율성이 중요하다.
