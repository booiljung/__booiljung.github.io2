프로세스 잡음 \mathbf w_k

프로세스 잡음 \mathbf{w}_k는 시스템 모델에서 실제 물리적 시스템의 불확실성이나 모델링 오류를 반영하기 위해 추가되는 항이다. 이는 시스템이 완벽하게 예측되지 않음을 나타내며, 주로 상태 전이 과정에서 발생하는 불확실성이나 외부 요인으로 인해 시스템의 상태가 예상과 다르게 변화할 수 있음을 의미한다.

프로세스 잡음 \mathbf{w}_k를 구성하는 방법은 문제의 성격에 따라 달라지지만, 일반적으로는 다음과 같은 과정을 통해 정의된다:

프로세스 잡음 모델링

1. 프로세스 잡음의 특성 설정

먼저, 프로세스 잡음이 어떤 특성을 가지는지 설정해야 한다. 프로세스 잡음은 보통 평균이 0이고, 정규 분포를 따르는 백색 잡음(white noise)으로 가정된다. 이때, 잡음의 분포는 다음과 같이 표현할 수 있다:

\mathbf{w}_k \sim \mathcal{N}(\mathbf{0}, \mathbf{Q}_k)

여기서 \mathbf{Q}_k는 프로세스 잡음의 공분산 행렬로, 이는 잡음의 크기와 각 상태 변수 간의 상관관계를 정의한다.

2. 프로세스 잡음 공분산 행렬 \mathbf{Q}_k 설계

프로세스 잡음의 공분산 행렬 \mathbf{Q}_k는 시스템에 따라 다르게 정의될 수 있다. 일반적으로 다음과 같은 방법으로 설계된다:

  • 경험적 방법: 특정 시스템의 특성을 분석하여 잡음의 분산(variance)과 상관관계(correlation)를 경험적으로 설정한다. 이는 실험 데이터나 도메인 전문가의 지식에 기반할 수 있다.

  • 단순화된 가정: 만약 시스템의 잡음 특성에 대한 정보가 부족하다면, 공분산 행렬을 대각 행렬로 가정하고 각 상태 변수에 대해 일정한 분산을 할당할 수 있다. 예를 들어, 단위 행렬 형태로 가정하여 각 상태 변수의 잡음이 독립적이고 동일한 분산을 가진다고 볼 수 있다:

\mathbf{Q}_k = \sigma^2 \mathbf{I}

여기서 \sigma^2는 잡음의 크기를 조절하는 파라미터이다.

  • 물리적 모델 기반 방법: 시스템이 물리적으로 설명될 수 있는 경우, 예를 들어 가속도의 변화나 외부 힘 등의 영향이 고려된 모델에서 파생된 공분산 행렬을 사용한다. 이 경우, 물리적인 동역학 모델에 따라 \mathbf{Q}_k를 더 정확하게 정의할 수 있다.

3. 프로세스 잡음의 추가

프로세스 잡음 \mathbf{w}_k는 앞서 정의한 공분산 행렬 \mathbf{Q}_k에 따라 생성된 랜덤 벡터로, 이는 상태 전이 방정식에 추가되어 시스템의 불확실성을 모델링한다. 이는 시스템의 동적 모델에 랜덤 요소를 추가하는 것과 같다. 예를 들어, \mathbf{w}_k를 랜덤 벡터로 정의하여 상태 전이 방정식에 추가한다:

\mathbf{x}_{k+1} = \mathbf{F}_k \mathbf{x}_k + \mathbf{B}_k \mathbf{u}_k + \mathbf{w}_k

이때 \mathbf{w}_k는 다음과 같이 생성할 수 있다:

\mathbf{w}_k = \mathbf{L}_k \mathbf{\eta}_k

여기서 \mathbf{\eta}_k는 평균이 0이고 공분산이 단위 행렬인 정규 분포를 따르는 랜덤 벡터, \mathbf{L}_k는 \mathbf{Q}_k의 Cholesky 분해로 구해진 하삼각 행렬이다. \mathbf{L}_k를 이용하면, \mathbf{w}_k의 공분산이 \mathbf{Q}_k가 되도록 보장할 수 있다.

이와 같이 프로세스 잡음 \mathbf{w}_k는 시스템의 불확실성을 반영하며, 이로 인해 칼만 필터가 현실 세계의 불확실성을 고려하여 보다 정확한 추정치를 제공할 수 있게 된다.
