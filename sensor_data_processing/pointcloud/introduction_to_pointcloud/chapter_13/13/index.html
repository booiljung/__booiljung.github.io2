포인트클라우드 세그먼테이션(Segmentation)은 3D 공간에서 스캔한 데이터를 의미 있는 부분으로 분리하는 작업이다. 세그먼테이션은 다양한 분야에서 활용되며, 주로 객체 인식, 로봇 내비게이션, 증강 현실, 그리고 자율 주행과 같은 애플리케이션에서 중요하다.

세그먼테이션 작업은 포인트클라우드에서 각 점의 속성을 기반으로 그룹화하거나, 주어진 기준에 따라 특정 영역을 분할하는 방식으로 이루어진다. 일반적으로는 포인트클라우드를 여러 개의 서브클라우드로 나누고, 각 서브클라우드는 하나의 물체 또는 표면을 나타내는 데 사용된다.

1. 세그먼테이션의 기본 정의

세그먼테이션은 입력 데이터 \mathbf{P} = \{ \mathbf{p}_1, \mathbf{p}_2, \dots, \mathbf{p}_N \}에서 각 포인트 \mathbf{p}_i를 속성에 따라 구분하는 것이다. 여기서 \mathbf{p}_i는 3차원 좌표 (x_i, y_i, z_i)로 표현된다. 세그먼테이션의 목표는 이 포인트들의 집합을 여러 개의 서브셋으로 나누는 것이다.

이를 수학적으로 표현하면, 세그먼테이션은 포인트 집합 \mathbf{P}를 k개의 서브셋으로 나누는 문제로 정의된다:

\mathbf{P} = \bigcup_{j=1}^{k} \mathbf{P}_j, \quad \mathbf{P}_i \cap \mathbf{P}_j = \emptyset \text{ for } i \neq j

각 서브셋 \mathbf{P}_j는 의미 있는 그룹, 예를 들어 평면이나 특정 물체의 표면과 같은 영역으로 정의된다.

2. 세그먼테이션 기법

포인트클라우드 세그먼테이션 기법은 크게 두 가지로 나눌 수 있다: 1. 기하학적 세그먼테이션 (Geometric Segmentation): 점 간의 기하학적 관계를 이용해 세그먼테이션을 수행한다. 2. 기능적 세그먼테이션 (Functional Segmentation): 포인트의 속성이나 외부 정보에 기반하여 세그먼테이션을 수행한다.

3. 기하학적 세그먼테이션

3.1. 유클리드 거리 기반 세그먼테이션

유클리드 거리 기반 세그먼테이션(Euclidean Cluster Extraction)은 포인트 간의 거리를 기준으로 클러스터를 형성하는 방법이다. 이 방식은 각 포인트 \mathbf{p}_i와 이웃한 포인트 \mathbf{p}_j의 거리가 임계값 \epsilon 이하일 때, 두 포인트를 같은 클러스터로 분류한다.

\text{if } \| \mathbf{p}_i - \mathbf{p}_j \| \leq \epsilon, \quad \mathbf{p}_i, \mathbf{p}_j \in \mathbf{P}_k

유클리드 거리 기반 세그먼테이션은 컴퓨팅 자원이 적게 들고, 특정 물체와 환경에서 빠르게 작동하지만, 복잡한 기하학적 구조에서는 한계가 있을 수 있다.

3.2. RANSAC 기반 평면 분할

RANSAC(Random Sample Consensus)은 포인트클라우드에서 특정 기하학적 모델(예: 평면, 원통)을 찾는 데 널리 사용되는 기법이다. RANSAC은 임의의 샘플을 추출하여 모델을 생성하고, 나머지 포인트들이 해당 모델에 얼마나 잘 맞는지를 평가한다. 평면 모델의 경우, 평면 방정식은 다음과 같다:

ax + by + cz + d = 0

여기서 a, b, c, d는 평면의 매개변수이고, 포인트 (x, y, z)는 평면 위에 있는 점이다. RANSAC을 통해 데이터 중에서 평면에 적합한 부분을 찾아 분할한다.

4. 기능적 세그먼테이션

4.1. 색상 기반 세그먼테이션

포인트클라우드에 색상 정보가 포함된 경우, 각 포인트의 색상 데이터를 이용해 세그먼테이션을 수행할 수 있다. 색상 기반 세그먼테이션은 색상 공간에서 거리 \Delta E를 계산하여 포인트들 간의 유사도를 판단한다.

색상 거리는 다음과 같이 계산된다:

\Delta E = \sqrt{(R_1 - R_2)^2 + (G_1 - G_2)^2 + (B_1 - B_2)^2}

여기서 R, G, B는 각 포인트의 RGB 값이다.

5. 세그먼테이션 알고리즘의 적용

PCL 라이브러리에서는 여러 가지 세그먼테이션 알고리즘을 제공한다. 그 중 대표적인 두 가지는 유클리드 거리 기반 클러스터링과 RANSAC 기반 평면 추출이다.

5.1. Euclidean Cluster Extraction in PCL

pcl::EuclideanClusterExtraction<pcl::PointXYZ> ec;
ec.setClusterTolerance(0.02);  // 2cm tolerance
ec.setMinClusterSize(100);
ec.setMaxClusterSize(25000);
ec.setInputCloud(cloud);
ec.extract(cluster_indices);

위 코드는 포인트클라우드에서 유클리드 거리 기반 클러스터를 추출하는 예시이다. 클러스터 간의 최소 거리는 2cm로 설정되었고, 각 클러스터는 최소 100개의 포인트를 포함한다.

5.2. RANSAC-based Plane Segmentation in PCL

pcl::SACSegmentation<pcl::PointXYZ> seg;
seg.setOptimizeCoefficients(true);
seg.setModelType(pcl::SACMODEL_PLANE);
seg.setMethodType(pcl::SAC_RANSAC);
seg.setDistanceThreshold(0.01);
seg.setInputCloud(cloud);
seg.segment(*inliers, *coefficients);

이 코드는 RANSAC을 사용해 포인트클라우드에서 평면을 분할하는 예시이다. 각 포인트가 평면에 속하는지 여부는 1cm의 거리 임계값을 기준으로 판단된다.

6. 포인트클라우드 세그먼테이션의 응용

세그먼테이션은 다양한 실세계 응용에서 중요한 역할을 한다. 예를 들어, 자율 주행 차량에서는 주위 환경을 포인트클라우드로 스캔하여 도로, 보행자, 장애물을 분리하는 데 사용된다. 이때 세그먼테이션을 통해 의미 있는 객체들을 인식하고, 이를 기반으로 경로를 계획하거나 충돌을 회피한다.

6.1. 객체 기반 세그먼테이션

객체 기반 세그먼테이션(Object-based Segmentation)은 포인트클라우드에서 특정한 객체만을 추출하는 작업이다. 이를 위해선 객체의 기하학적 특성, 색상, 혹은 거리 정보를 이용할 수 있다.

예를 들어, 차량 주차장의 포인트클라우드에서 차량만을 추출하기 위해서는 차량의 크기, 형상, 그리고 상대적인 높이 등의 정보를 기반으로 세그먼테이션을 수행할 수 있다.

6.2. 포인트클라우드 데이터에서 벽과 바닥 분리

건축 구조물의 포인트클라우드에서는 벽과 바닥을 분리하는 것이 중요한 세그먼테이션 작업 중 하나이다. 이를 위해선 각 포인트의 기울기나 높이 정보를 이용하여 구분할 수 있다. PCL 라이브러리에서 RANSAC을 사용해 평면(바닥)을 검출한 후, 나머지 포인트들을 이용해 벽을 추출하는 방법이 자주 사용된다.

6.3. 클러스터 기반 객체 검출

포인트클라우드에서 각각의 객체를 독립적으로 인식하는 클러스터 기반 객체 검출은 유클리드 거리 기반 클러스터링을 활용하여 이루어진다. 예를 들어, 주차장에서 여러 차량이 인접해 있는 상황에서, 각 차량 간의 거리를 계산하여 이를 분리하는 방식이다.

다음은 이와 관련된 예시 코드이다:

pcl::search::KdTree<pcl::PointXYZ>::Ptr tree (new pcl::search::KdTree<pcl::PointXYZ>);
ec.setSearchMethod (tree);
ec.setInputCloud (cloud);
ec.extract (cluster_indices);

이 코드는 KdTree를 사용해 포인트클라우드에서 객체를 클러스터링하는 과정이다. 각 클러스터는 독립적인 객체로 처리된다.

7. 세그먼테이션의 평가 지표

세그먼테이션 결과를 평가하기 위한 다양한 지표들이 존재한다. 특히 세그먼테이션의 정확도와 효율성을 평가하는 것이 중요하다.

7.1. 정확도 평가

정확도는 주로 정답(Ground Truth) 세그먼트와의 비교를 통해 평가된다. 포인트클라우드 세그먼테이션에서는 주로 Precision, Recall, Intersection over Union (IoU)와 같은 지표가 사용된다.

  • Precision: 검출된 포인트 중 실제 객체에 속하는 포인트의 비율
\text{Precision} = \frac{TP}{TP + FP}
  • Recall: 실제 객체에 속하는 포인트 중에서 검출된 포인트의 비율
\text{Recall} = \frac{TP}{TP + FN}

여기서 TP는 True Positive, FP는 False Positive, FN은 False Negative를 의미한다.

7.2. Intersection over Union (IoU)

세그먼테이션 결과의 품질을 평가하는 데 중요한 또 다른 지표는 IoU이다. IoU는 검출된 영역과 실제 영역 간의 겹치는 부분을 전체 영역과 비교한 값으로 정의된다.

\text{IoU} = \frac{\mathbf{P}_{\text{intersection}}}{\mathbf{P}_{\text{union}}}

IoU가 1에 가까울수록 세그먼테이션 결과가 정확하다는 의미이다.

8. 포인트클라우드 세그먼테이션의 한계와 문제점

세그먼테이션은 매우 유용한 기술이지만, 실제 응용에서 몇 가지 한계를 가지고 있다.

8.1. 데이터의 복잡성

포인트클라우드 데이터가 복잡해질수록 세그먼테이션의 난이도도 높아진다. 특히 복잡한 실내 환경이나 고밀도의 3D 스캔에서는 객체들 간의 경계가 모호해져 정확한 세그먼테이션이 어려워질 수 있다.

8.2. 노이즈와 이상치의 처리

노이즈나 이상치가 포함된 데이터는 세그먼테이션 성능을 저하시킬 수 있다. 특히 RANSAC과 같은 기법은 노이즈에 민감하기 때문에, 노이즈 제거 전처리 과정이 필수적이다.

이상치는 일반적으로 포인트클라우드의 경계나 센서 오차로 인해 발생하며, 이를 효과적으로 처리하지 않으면 잘못된 세그먼테이션 결과를 초래할 수 있다.
